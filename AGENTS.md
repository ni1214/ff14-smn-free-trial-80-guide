# 開発ルール / AI作業メモ

このリポジトリは、GitHub Pagesで公開しているFF14フリートライアルLv80向けのスマホ用攻略アプリです。

公開URL:

- https://ni1214.github.io/ff14-smn-free-trial-80-guide/

## 基本方針

- すべて日本語UIにする。
- スマホ縦画面を最優先にする。
- ログイン機能は入れない。二人用だが、進捗共有機能も入れない。
- 保存はスマホごとの `localStorage` を基本にする。
- 外部リンクは日本語ページを優先する。
- FF14のスキル名、アイコン、習得Lvは公式ジョブガイド準拠にする。
- 装備名と装備アイコンはできる限り公式DB、公式アイコン、既存JSONに合わせる。
- 低レベル帯でも「どこで何を揃えるか」がアプリ内で完結するようにする。
- 詳細情報は必要な場所に折りたたみ、初心者が最初に見る情報は少なくする。

## 現在の主な構成

- `index.html`: 画面、スタイル、主要ロジックを含む単一HTML。
- `action-icons.json`: 既存アクションアイコン補助データ。
- `job-actions.json`: 公式ジョブガイド由来の全ジョブアクション、習得Lv、アイコン、クエスト要否データ。
- `gear-catalog.json`: 装備候補データ。
- `item-icons.json`: アイテムアイコン補助データ。
- `.nojekyll`: GitHub Pagesでそのまま静的配信するために必要。

## UIルール

- ボトムナビは増やしすぎない。
- 現在のタブ構成:
  - 装備/入手法
  - スキル/成長
  - 楽しみナビ
  - 初見ギミック
  - リンク/出典
- 「迷子防止メモ」と「ごほうび候補」は楽しみナビ内に統合済み。
- 「キャラ確認」タブは廃止済み。Lodestone自キャラリンクは上部の「マイキャラ」に置く。
- 「マイキャラ」にはジョブ、レベル、メイン進行、詩学、自キャラLodestoneリンクを集約する。

## 実装ルール

- 変更はなるべく `index.html` と既存JSONに閉じる。
- 新しいライブラリやビルド環境は増やさない。
- FirebaseやDBは現在未使用。ユーザーが明確に必要と判断するまでは追加しない。
- 公式アイコンを使う機能では、画像URLと表示名の不一致を避ける。
- レベル別スキル回しは、現在Lvで未習得のボタンを表示しない。
- 全スキル一覧では未習得スキルを薄く表示し、習得Lvと必要クエスト情報を出す。
- 公開前に最低限、HTML内JavaScript構文とJSON構文を確認する。

## 公開ルール

GitHub Pagesは `main` ブランチへpushされると自動で公開されます。

通常の公開手順:

```powershell
.\tools\publish.ps1 -Message "変更内容を短く書く"
```

手動で行う場合:

```powershell
git status --short
git add -A
git commit -m "変更内容を短く書く"
git push origin main
```

公開後は数十秒待ってから、公開URLで反映確認します。
